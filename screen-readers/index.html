Demo Page for Screen-Readers Not Reading Spaces

This is a proof-of-concept page showing issues in NVDA and VoiceOver. In NVDA, spaces are not read, and in VoiceOver, words are read as single characters at a time. Some hidden text as an example:

This has no spaces in NVDA and as single characters in VoiceOver

This appears to be caused by adding word-wrap: break-word; to a parent element. Applying white-space: nowrap; or word-wrap: normal seems to fix the issue. For example:

Using white-space: nowrap, NVDA separates the words, and VoiceOver reads as a sentence.
With "word-wrap: normal", NVDA reads this fine, but VoiceOver does so as a series of single words.

Not using word-wrap: break-word; at all produces something like this:

NVDA has no issues, but VoiceOver still reads as individual words.

My recommendation would be white-space: nowrap as it fixes the issues in both NVDA and VoiceOver, regardless of whether word-wrap was used on a parent element.
